# UI 设计

> 核心目标：输出**可视化设计文档**，让开发者和用户一眼看懂界面结构。适用于 Web 应用、Qt Widget、Qt QML 等桌面/移动端界面。

## 执行流程

```
需求理解 → 信息架构 → 页面拆分 → 流程梳理 → 可视化设计 → 白话说明 → 按需展开
```

---

### 1. 需求理解

**输入源**：PRD 文档 或 用户口述需求

**必确认的 6 个问题**：

| # | 问题 | 作用 |
|---|------|------|
| 1 | 产品类型？（工具 / 内容 / 电商 / 数据 / 管理后台） | 决定布局风格 |
| 2 | 目标用户？（专业人员 / 普通消费者 / 内部运营） | 决定信息密度 |
| 3 | 核心任务？（用户主要完成什么，高频操作路径是什么） | 决定功能优先级 |
| 4 | 平台类型？（Web / Qt Widget / Qt QML / 移动端） | 决定布局体系 |
| 5 | 现有材料？（口述 / 功能清单 / PRD / 草图） | 决定从哪一步开始 |
| 6 | 有无参考产品？ | 决定风格参考 |

> **用户只有模糊需求时，重点追问 4 个问题**：
> 1. 用户打开界面，第一眼需要看到什么？
> 2. 最高频的操作是什么？要放在哪里最容易点到？
> 3. 哪些信息需要同时对照查看？（决定要不要分屏）
> 4. 数据量有多大？（决定用卡片、表格还是列表）

---

### 2. 信息架构

画布局前先梳理**功能清单**，确保没有遗漏：

```markdown
| 功能模块 | 子功能 | 优先级 | 说明 |
|---------|--------|--------|------|
| 用户管理 | 登录/注册 | P0 | 手机号+验证码 |
| 数据展示 | 仪表盘 | P0 | 核心数据概览 |
```

**功能→区块映射**：把功能翻译成界面区块

| 功能特征 | 界面区块 | 布局位置 |
|---------|---------|---------|
| 全局导航 | 导航栏、Sidebar | 左侧或顶部 |
| 核心数据 | 仪表盘、KPI 卡片 | 首屏最显眼位置 |
| 搜索/筛选 | 搜索框、筛选面板 | 内容区上方 |
| 数据列表 | 表格、卡片列表 | 主内容区，最大面积 |
| 高频操作（增删改） | 工具栏、悬浮按钮 | 内容附近 |
| 辅助信息/提示 | 侧边面板、Toast | 边缘，不遮挡主内容 |
| 需要对比的数据 | 分屏、双表格 | 左右布局 |

**推导原则**：全局框架 → 核心内容区 → 操作区 → 辅助区

---

### 3. 页面拆分

判断需要几个页面/窗口：

| 判断依据 | 拆分方式 | 示例 |
|---------|---------|------|
| 功能独立性高 | 独立页面/窗口 | 登录页 vs 主界面 |
| 同一任务流 | Tab 切换 | 设置页的多项 Tab |
| 信息密度大 | 分步向导 | 复杂表单的 3 个步骤 |
| 需要对比参照 | 分屏/双窗口 | 代码编辑器左右对比 |
| 模态操作 | 弹窗/对话框 | 确认删除、快速编辑 |

**输出**：页面清单，标注类型（独立页面 / Tab / 弹窗 / 侧边面板）

---

### 4. 流程梳理（多页面时必做）

先画页面流转关系，再画每个页面的详细布局：

```markdown
[登录页] --登录成功--> [仪表盘首页]
                      |
                      ├--点击导航--> [数据管理页]
                      |                    |
                      |                    ├--点击新增--> [新增弹窗]
                      |                    └--点击详情--> [详情面板]
                      |
                      └--点击设置--> [设置页]
```

### 5. 可视化设计（必须输出）

**每个页面/窗口必须有 ASCII 布局图 + 区块说明**。输出到 `docs/design/[module]-ui-design.md`。

#### 示例：Web 管理后台首页

```
┌─────────────────────────────────────────────┐
│  Header (64px)                              │
│  Logo | 搜索           [通知] [头像]         │
├──────────┬──────────────────────────────────┤
│          │                                  │
│  Sidebar │  Main Content                    │
│  (200px) │  ┌────────────────────────┐      │
│          │  │     数据卡片区域        │      │
│  导航     │  └────────────────────────┘      │
│          │  ┌────────────────────────┐      │
│          │  │     图表区域           │      │
│          │  └────────────────────────┘      │
└──────────┴──────────────────────────────────┘
```
- **Header**: 固定顶部，64px，含 Logo、搜索、通知、头像
- **Sidebar**: 左侧导航，200px，可折叠
- **Main**: 自适应，最大 1400px，24px 内边距

#### 示例：Qt Widget 主窗口

```
┌─────────────────────────────────────────────┐
│  Menu Bar                                   │
├─────────────────────────────────────────────┤
│  Tool Bar (48px)                            │
├──────────┬──────────────────────────────────┤
│          │                                  │
│  Dock    │  Central Widget                  │
│  (240px) │  ┌────────────────────────┐      │
│  项目树   │  │     属性编辑器          │      │
│          │  └────────────────────────┘      │
│          │  ┌────────────────────────┐      │
│          │  │     预览/画布区域       │      │
│          │  └────────────────────────┘      │
├──────────┴──────────────────────────────────┤
│  Status Bar                                 │
└─────────────────────────────────────────────┘
```
- **Menu Bar**: 菜单栏（文件 / 编辑 / 视图）
- **Tool Bar**: 常用操作快捷入口
- **Dock**: 可浮动/停靠的侧边面板
- **Central Widget**: 核心工作区
- **Status Bar**: 状态栏

**布局图绘制步骤（按顺序）**：

1. **画骨架**：最外层框架（Header/Sidebar/主内容区）
2. **分区块**：主区域内划分功能区块，标注名称
3. **填元素**：每个区块画 1-3 个关键元素（搜索框、表格、按钮）
4. **标尺寸**：高度/宽度/间距
5. **写说明**：每个区域放什么、用户做什么

**可视化强制要求**：
- ❌ 禁止只写文字描述，必须有 ASCII 图
- ✅ 标注区域名称、关键尺寸
- ✅ 每个区块说明：放什么内容、用户做什么操作

### 6. 白话说明（必须输出）

**每个页面布局图下面，必须配一段大白话**，让不懂技术的人也能看懂：

> **写法要点**：
> - 不用术语，用"点击"、"看到"、"弹出"、"跳转到"
> - 按用户视角写："用户一打开页面，首先看到..."
> - 明确每个按钮/链接点击后发生什么
> - 说明弹窗、跳转、加载等效果

**示例**：

```markdown
### 白话说明：设备管理页

用户一打开这个页面，左边看到设备列表，右边看到第一个设备的详细信息。

- **点左边列表里的设备**：右边详情跟着变，显示这个设备的具体参数
- **点右上角"新增设备"按钮**：弹出一个填写框，让用户输入设备名称和编号，填完点"保存"就加到列表里
- **点每行后面的"编辑"**：详情区变成可编辑状态，改完点"保存"
- **点"删除"**：先弹确认框"确定要删除吗？"，点了"确定"才真删掉
- **在上方搜索框里打字**：下面的列表实时过滤，只显示匹配的设备
- **点"导出"按钮**：自动生成一个 Excel 文件下载到电脑
```

**必填要素检查**：
- [ ] 页面打开时默认显示什么？
- [ ] 每个按钮/链接点击后发生什么？（跳转 / 弹窗 / 提示）
- [ ] 有没有确认步骤？（删除、修改等危险操作）
- [ ] 搜索/筛选怎么工作？
- [ ] 数据量大的时候怎么展示？（分页 / 滚动 / 懒加载）

---

### 7. 什么时候不用

- 纯后端/工具开发，没有界面
- 用户只要求写代码，没提设计需求
- 已有设计系统/组件库，只需按规范调用
- 用户已提供完整设计稿（Figma / PSD / Qt Designer）

---

### 8. 按需展开

用户要求详细说明时，补充：
- 组件规范（Web 组件 / Qt 控件 / QML 元素）
- 交互状态（正常 / 悬停 / 按下 / 禁用 / 加载中）
- 响应式/窗口缩放适配
- 键盘导航与快捷键
- 深色/浅色模式
- 空状态（无数据 / 加载失败 / 权限不足）
- 错误提示方式（Toast / 弹窗 / 内联提示）

## 知识库

### 布局原则
- **Web**: 8px 网格；F 型/Z 型阅读
- **Qt Widget**: 善用 `QSplitter`；Dock 窗口灵活布局
- **Qt QML**: 用 Anchor/Layout；避免硬编码像素，用 `dp`

### 视觉层级
- 对比：字号/颜色/粗细至少一个维度有明显差异
- 留白：Web 区块间距 ≥ 48px，组件间距 16-24px；桌面控件间距 8-16px
- 对齐：统一左对齐或居中对齐，不要混用

### 颜色与字体
- 主色 1 个（CTA），中性色（灰阶），功能色（红/黄/绿）
- 最多 3 级字号：标题 / 正文 / 辅助
- **Qt**: 颜色用 `QPalette` 或主题统一管理，禁止硬编码 `#RRGGBB`
- **Qt 中文字体**：优先 `"Microsoft YaHei"`、`"PingFang SC"`、`"Noto Sans CJK SC"`
