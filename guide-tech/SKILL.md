# 技术讲解

## 核心原则

### 四层次讲解法

用**同一个生活例子**贯穿四层，由浅入深：

```
第 1 层: 生活类比 (What)  → 快递驿站、餐厅点餐
第 2 层: 概念映射 (How)   → 快递员 = 网络请求
第 3 层: 原理机制 (Why)   → HTTP 请求生命周期
第 4 层: 代码实现 (Code)  → 逐行解释（按需）
```

### 可视化优先

| 场景 | 方式 |
|------|------|
| 流程/步骤 | 箭头流程图 `A → B → C` |
| 层级关系 | 缩进树状图 |
| 对比关系 | 表格对照 |
| 数据流向 | 管道示意图 |

## 讲解模板

```markdown
## 一句话总结
[通俗概括核心概念，≤30 字]

### 第 1 层: 生活类比
> 场景: [生活场景]

### 第 2 层: 概念映射
| 生活中的 | 技术中的 | 作用 |
|---------|---------|------|

### 第 3 层: 原理机制
步骤1 → 步骤2 → 步骤3

### 第 4 层: 代码实现（按需）
[代码片段 + 逐行说明]
```

## 常用类比

| 技术概念 | 生活类比 |
|---------|----------|
| 客户端-服务器 | 顾客-餐厅 |
| 数据库 | 图书馆 |
| 缓存 | 书桌/口袋 |
| 消息队列 | 快递柜 |
| 负载均衡 | 多窗口排队 |
| 依赖注入 | 外卖点餐 |
