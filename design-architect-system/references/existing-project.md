# 老项目增量架构设计

用于在现有代码库里增加功能、改模块或做局部重构。重点是融入现有架构、最小改动、降低回归风险。

## 设计流程

1. 扫描现有代码：目录结构、构建系统、核心模块、已有测试、配置和文档。
2. 识别现有风格：分层方式、命名、依赖方向、错误处理、日志、测试方式。
3. 明确增量需求：新增什么行为，影响哪些用户场景。
4. 找复用点：优先复用现有模块和扩展点。
5. 找改动点：只列必要修改，避免重塑全局架构。
6. 设计新增模块：放入现有目录体系，不新开一套结构。
7. 标注影响面：API/ABI、配置、数据格式、线程、资源、测试、部署。
8. 给出回归策略：现有测试、新增测试、手动验证和风险兜底。

## 老项目防破坏检查

1. 是否遵守现有目录结构和命名风格？
2. 是否复用了已有模块，而不是另起炉灶？
3. 是否最小化修改公共接口？
4. 是否说明 API/ABI、配置、数据格式和序列化影响？
5. 是否说明线程、生命周期、资源释放影响？
6. 是否有现有回归测试和新增测试切入点？
7. 是否有回退方案？
