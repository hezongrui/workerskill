# 新项目架构设计

用于从零设计一个新项目。重点是从业务场景推导最小可落地架构，不按模板堆组件。

## 设计流程

1. 明确目标：一句话说明系统要解决的问题。
2. 列核心场景：2 到 5 个主流程，按 P0/P1/P2 排序。
3. 列约束：技术栈、平台、部署、团队规模、性能、可靠性、安全、周期。
4. 判断复杂度：小工具、单模块应用、多模块产品、平台级系统。
5. 选择最小架构：先模块化单体，只有明确理由才拆服务或引入复杂组件。
6. 定义模块边界：每个模块一句话职责，明确输入、输出、依赖。
7. 记录关键决策：用 ADR-lite 表格写选择和后果。
8. 给出落地顺序：先核心闭环，再异常路径，再扩展能力。

## 新项目防过度设计检查

1. 能不能一个进程解决？
2. 能不能一个数据库或文件解决？
3. 能不能先不用缓存？
4. 能不能先不用消息队列？
5. 能不能先不用插件化？
6. 能不能先做 P0 主流程，再补扩展？

如果答案是“能”，默认采用简单方案。
