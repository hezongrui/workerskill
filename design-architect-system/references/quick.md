# Quick 模式

目标：快速交付，验证想法。
原则：能简则简，不过度设计。

## 架构特征

- 模块数：1-3 个
- 分层：无，按功能组织
- 存储：单数据库或文件
- 部署：单容器或直接运行

## 4. 目录结构

```
project/
├── src/                    # 源码根目录
│   ├── main.py             # 应用入口
│   └── utils/              # 工具函数
├── tests/                  # 测试文件
├── config/                 # 配置文件
└── README.md               # 项目说明
```

| 目录/文件 | 功能说明 |
|-----------|----------|
| `src/` | 所有业务代码 |
| `src/utils/` | 通用工具、辅助函数 |
| `tests/` | 单元测试与集成测试 |
| `config/` | 环境配置、常量定义 |

| 风险 | 触发条件 | 影响 | 应对 | 回退 |

## 设计原则

1. 一个文件一个职责
2. 环境变量 > 配置文件
3. 先跑起来，不优化
4. 重复3次以下不抽象
5. 不预留"以后可能用"的接口
